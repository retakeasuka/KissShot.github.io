C++函数调用过程

C++函数调用过程

先看一段代码

#include<stdio.h>
int Add(int x, int y)
{
    int z = 0;
    z = x + y;
    return z;
}
#include <stdio.h>
int main()
{
    int a = 10;
    int b = 20;
    int c = Add(a, b);
    return 0;
}

先开始最为简单的解释

  • 上述代码中从main函数开始，从上到下，当main函数运行完之后，这个程序就运行结束了
  • 中间如果有子函数，运行子函数，子函数结束后，从子函数的位置再往下运行。
  • 即运行顺序为
    - int mian->
    - int a->
    - int b->add->
    - int z->
    - z=x+y->
    - return->
    - int c->
    - return

这一部分的内容应该就是属于常识了

内存中是如何调用的

前置

  1. 首先我们需要了解到的是简单的内存情况，内存是连续分布的一块地址，而上面的例子，就是在划分的一个内存地址块中，这块地址在被称为“栈区“。
  2. 简单的介绍一下CRTStartUp函数，这个是程序的真正入口，对程序员是透明的，由编译器链接libcd.lib里面的函数达成。
  3. 最为关键的指针/寄存器，ebp（栈底）,esp（栈顶）
    接下来的介绍就不从汇编语言和硬件基础开始介绍了，这部分我们直接忽略，如果有兴趣可以参见其他大佬们的文章，写的很清楚了，本文仅介绍核心的逻辑。

堆栈的调用

  1. 创建Main函数，通过把eso放在低位地址，空位添加cccccc，以及一些寄存器的信息（这里接直接忽略掉了哦），得到一个esp~ebp的空间，抽象出来就是我们的main函数了。（也就是main函数实际长的样子）

    此时声明a,b函数就是放在了栈的底部。（注意，a,b并不是通过pop,pus进去的，所以在底部，实际上是先初始化了很多东西才是吧a,b整进去的，可以想想这个的生命周期）
  2. 调用Add函数：（也就是汇编中的call，更深一点就是jump指令），这个函数的创建过程，可以认为是和Main差不多，把Main函数当做CRTStartUp，重复上面的创建过程

    然后声明z=a+b，存在一个寄存器里面，跟随返回值回去，也就是常见的return xxx。
  3. 接下来就是销毁这个Add函数，销毁的过程就是把esp移动带ebp那，上面的值就不知道是啥了，也不知道位置什么的。只留了一个返回值
  4. 同样，Main函数继续往下执行，执行到最后，需要进行程序的退出，退出之前，还需要把Main函数给销毁掉。过程和销毁Add函数比较相似，不过会多一步，就是有一个cmp操作（计算机最基本的指令，比较两个数的值是否相等），如果相等的话就退出程序。

总结

整个调用过程实际上就是通过最为关键的esp,ebp来进行创建内存空间和销毁存储空间，当ebp!=esp的时候，就是有一个函数空间，当ebp=esp的时候，就是某某空间被创建后销毁了。
