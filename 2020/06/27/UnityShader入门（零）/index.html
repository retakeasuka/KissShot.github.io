一、Shader概述

1. 什么是Shader？

我最简单的一个回答就是，着色器

2. 渲染流水线

2-1. 什么是渲染流水线？

Cpu和Gpu之间的通信

2-2. 干了什么

2-2-1. 把数据加载到显存中

即所有渲染的数据，硬盘->内存->显存，因为显卡对显存的访问最快

2-2-2. 设置渲染状态

定义了这些网格是怎么样被渲染的

2-2-3. 调用DrawCall

调用Gpu的流水线


enter description here

2-3. 我们能在这个过程中干什么

这个过程中，有些部分我们拥有绝对的掌控权，即完全可编程，有些部分我们可以处理部分数据，做一些操作，而有些部分，则是我们不可操作的，完全由计算机来操作

2-3-1. 顶点着色器

  1. 完全可编程
  2. 处理单位是顶点，也就是每个顶点都会调用一次顶掉着色器
  3. 作用是坐标变换和逐顶点的光照，还输出后续的数据
    • 一个最简单的顶点的着色器的工作就是把顶点坐标从模型空间转换到齐次裁剪空间（后面数学部分会介绍）
  4. 特点是，顶点着色器不能创建删除顶点，也不能获取顶点之间的关系，正是因为这样，GPU利用自己的特点，并行化处理，速度就非常快

2-3-2. 裁剪

  1. 作用就是过滤掉视锥外面的点（后面会有视锥的介绍）
  2. 无法控制，但是可以自定义一个裁剪操作来配置

2-3-3. 屏幕映射

  1. 作用是，把三维立方体下面的坐标转换到屏幕坐标系（降维度）

2-3-4. 三角形设置

  1. 标志：这一步就进入了光栅化的阶段 [https://baike.baidu.com/item/光栅化]
  2. 计算一个三角网格所需要的信息，具体一点就是，上一个阶段传入的是三角网络格的顶点，我们需要计算一些，列如三角形的方程，边的一些信息。就是提供下一步进行三角形遍历给的一些生成的数据

2-3-5. 三角形遍历

  1. 作用：会检查每个像素是否被一个三角网格所覆盖，如果被覆盖就会生成一个片元。
  2. 插值[https://baike.baidu.com/item/插值/13014727]
  3. 注意的地方：一个片元不是一个像素，只是一个包含了各种信息的集合。

2-3-6. 片元着色器

  1. 可编程
  2. 输出一个或者多个着色器
  3. 只能影响单个的片元

2-3-7. 多片元操作

  1. 冲冲冲，输出阶段
  2. 决定了我是不是能够看到这个片元，要经过很多测试
  3. 混合

2-3-8. 一些问题

2-3-8-1. OpenGL/DirectX到底是个啥？

在硬件层上面的一个抽象，方便和GPU打交道

二、Unity Shade 概述

1. 材质和Unity Shader

1-1. 如何在UNITY里面写一个Shader

  1. 新建材质
  2. 新建shader
  3. 材质选择shader
  4. 新建实体，选择材质

2. shaderlab

基础结构

Shader"My Shader"{
	properties{
		//可见显示组件
	}
	SubShader{
		//显卡使用过的着色器
		pass{
				//每个pass代表一个完整的渲染流程
		}
	}
		//Shader失败之后的保险
	Fallback "vertexlit"
}

2-1. properties

设置各种属性

2-2. subshader

定义了一系列的PASS以及可选的状态和标签设置，每个pass定义了一次完整的渲染流程

2-3. pass

因为每个pass定义了一次完整的渲染流程
尽量减少调用pass,不然会造成渲染性能的下降
我们在subshader进行的设置会影响到所有Pass

2-4. rendersetup

作用，提供一些列渲染给出的设置指令，可以设置显卡的各种状态，比如是否开启混合，深度之类的

2-5. tags

告诉Unity的渲染引擎，SubShader是怎么希望的，什么时候渲染这个对象

2-6. fallback

渲染失败之后默认给一个默认的材质，以防爆炸
